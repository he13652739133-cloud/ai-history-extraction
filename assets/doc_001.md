# AI 处理史料讲座：提示词合集

## 1. 判断史料类型

你是一名史料数字化整理助手。请先判断我提供的材料属于哪一类：
1. 可检索文字 PDF；
2. 扫描版 PDF 或图片；
3. 疑难文字材料，例如手写、碑刻、竖排、繁体、异体字、磨损材料；
4. 图像史料；
5. 录音或口述史材料；
6. 整本书或大规模文本。

请输出：材料类型、可采用的识别方法、可能风险、下一步处理建议。不要直接做历史解释。

---

## 2. OCR / VLM 转录

请作为严谨的史料转录助手识别这份材料。规则如下：
1. 只根据图像或原文内容转录，不要自行补字。
2. 无法辨认的字用 □ 表示。
3. 不确定的字后加 ？。
4. 尽量保留原有行序、段落和页码。
5. 在最后列出“需人工复核清单”，说明哪些字、词、行、字段需要复核。

输出格式：
- 材料概况
- 逐行转录
- 疑难字表
- 需人工复核清单

---

## 3. 史料结构化抽取

你是一名严谨的史料整理助手。请只根据我提供的原文/图片内容抽取信息，不要自行补全。

任务：把材料整理为结构化表格。
字段包括：编号、材料类型、人物/机构、时间、地点、事件、金额/数量、关系、原文证据、页码或图片位置、置信度、需人工复核的问题。

规则：
1. 原文没有的信息填“未见”。
2. 不能确定的字用 □ 表示，并在“需人工复核的问题”中说明。
3. 每一行必须保留至少一段原文证据。
4. 不要把推测写成事实；推测只能放在备注中。
5. 输出 Markdown 表格，同时再输出 JSON 数组，方便后续转 Excel。

---

## 4. 墓碑图像识别

请作为史料整理助手识别这张墓碑图像。不要自行编造看不清的信息。

请分四部分输出：
1. 图像概况：墓碑类型、碑面状态、文字方向、是否有遮挡/磨损。
2. 逐行转录：尽量按碑面原有行序输出；无法辨认的字用 □；不确定的字后加“？”。
3. 字段抽取：墓碑编号、姓名/碑名、英文名、性别、类型、出生年、卒年、籍贯、建立者、建立者关系、碑形、装饰、位置、是否完整。
4. 复核清单：列出必须人工检查的字、行、字段和原因。

输出要求：每个字段附置信度：高 / 中 / 低。

---

## 5. 侨批类材料抽取

请把以下侨批材料整理成结构化数据。只抽取原文明确出现的信息。

字段：
编号、写信人、收信人、写信地点、收信地点、日期、金额、币种、汇款用途、亲属关系、情绪表达、事件类型、原文证据、页码、置信度、疑难点。

规则：
1. 金额和币种必须保留原文写法，同时在备注中给出标准化结果。
2. 亲属关系不能凭常识推断，必须有原文证据。
3. 情绪表达只做初步标注，例如思念、焦虑、嘱托、报平安、经济压力。
4. 输出表格，并列出人工复核清单。

---

## 6. 家谱 / 族谱关系抽取

请把以下族谱材料整理为人物关系数据。只抽取原文明确出现的信息。

输出两张表：
A. 人物表：person_id、姓名、字/号、性别、生卒年、籍贯、所属世代、原文证据、出处。
B. 关系表：source_person、target_person、关系类型、关系说明、原文证据、出处、置信度。

关系类型优先使用：父子、母子、夫妻、兄弟、迁徙、任职、居住地、墓葬地。

规则：
1. 同名人物不要直接合并。
2. 世代关系不清楚时标记为“待复核”。
3. 每条关系必须有原文证据。

---

## 7. 整本书转知识图谱

请把以下历史文本转为知识图谱数据。只抽取原文明确出现的信息，不要补充常识。

输出两张表：
A. 节点表：node_id、节点名称、节点类型（人物/地点/机构/事件/文献）、原文证据、出处。
B. 关系表：source_node、target_node、关系类型、关系说明、原文证据、出处、置信度。

关系类型优先使用：亲属、任职、隶属、同盟、敌对、师生、迁徙、发生于、参与、记载于。

质量要求：
1. 每条边必须有原文证据。
2. 不确定关系标记为“待复核”。
3. 同一人物有异名时，单独列出“可能同一实体”的说明，不要直接合并。

---

## 8. GPT 复核表格

请作为审校助手检查这份史料结构化表格。不要新增史实，只做质量检查。

请检查：
1. 是否有空缺字段；
2. 是否有日期、年龄、生卒年逻辑矛盾；
3. 是否有人名、地名、机构名写法不一致；
4. 是否有金额、数量异常值；
5. 每行是否有原文证据；
6. 哪些记录需要人工回看原图或原文。

输出：问题类型、行号、字段名、问题说明、修改建议、是否必须人工复核。
